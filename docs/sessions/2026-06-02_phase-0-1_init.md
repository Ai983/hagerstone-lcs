## Deviations from the PRD / prompt (and why)
1. **No token-handoff `/auth/callback` in practice.** The prompt's Phase 1.4 assumed the Hub hands tokens to sub-apps. The live Hub does NOT (just `window.open`); each sub-app logs in independently. Faithful match = CPS-style independent login. Built that, plus a forward-compatible `/auth/callback` for the planned SSO. User confirmed 2-login approach for now.
2. **Schema exposure + Vercel deploy are now DONE** (resolved within this session — see 0.3/1.7 above). The **only remaining item is redeploying the Hub portal** so the LCS tile appears; the tile URL already matches the live deploy.
3. **Stack: React 18 + Vite 5 + Tailwind 3.4** instead of CPS's React 19 / Vite 8 — to avoid local Node-toolchain friction. Same stack contract (React+TS+Vite, Tailwind v3, shadcn, TanStack v5, RR, RHF+Zod, Lucide, Sonner) and build script `vite build`.

## Post-audit fix (same session)
- **Bug:** on load, the "No access to LCS" screen flashed for ~0.2s before the correct gated home rendered. **Cause:** `AuthContext` resolved the session from two concurrent paths (`getSession()` + `onAuthStateChange`); `ProtectedRoute` evaluated the gate mid-race, when `loading` had flipped false but `employee` was momentarily null. **Fix:** rewrote `AuthContext` to a single sequence-guarded resolver (`seqRef`) that applies `employee`/`role`/`hasLcsAccess`/`loading` atomically at the end, drops stale/duplicate resolutions, and defers DB calls off the `onAuthStateChange` tick (avoids the supabase auth-lock deadlock). Rebuilt + redeployed LCS (READY). No DB changes.
